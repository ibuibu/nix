# スマートコミット

現在のステージングされた変更を分析して、コミットメッセージを生成し、承認されたらコミット＆プッシュを行います。

手順：
1. `git diff --staged`で変更内容を確認
2. `git status`で現在の状態を確認
3. `git log --oneline -5`で最近のコミットメッセージのスタイルを確認
4. 上記の情報を基に、適切なコミットメッセージを生成（conventional commit形式推奨）
5. 生成したメッセージを明確に表示し、「このメッセージでコミットしてよろしいですか？ (y/n)」と承認を求める
6. ユーザーが「y」または「yes」で承認した場合のみ、以下を実行：
   - `git commit -m "生成したメッセージ"`でコミット
   - リモートにプッシュするか確認し、承認されたら`git push`を実行
7. ユーザーが「n」または「no」と答えた場合は、コミットをキャンセル

重要な注意事項：
- 実際にコミットする前に、必ずユーザーの明示的な承認（「y」または「yes」の返答）を待つこと
- 承認なしに勝手にコミットしないこと
- コミットメッセージにはフッターを付けず、シンプルなメッセージのみを使用すること
