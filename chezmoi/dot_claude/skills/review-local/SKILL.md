# review-local

CI環境でローカルのコード変更をレビューする。

> **注意**: このスキルはCI（非対話）用。CLAUDE.md の「PRレビュー」セクション（ユーザーと1件ずつ相談するフロー）は適用しない。

## 言語

出力はすべて**日本語**で記述する。コードの識別子や技術用語は英語可。

## 手順

### 1. コンテキスト収集

- リポジトリルートの `CLAUDE.md` を読む
- `.claude/rules/` があればルールファイルをすべて読む
- 変更ファイルに関連する `*.md` ドキュメントのみ読む（不要なものは読まない）

### 2. 変更差分の取得

mainブランチとの差分を取得し、変更の目的とスコープを理解する。

```bash
git diff main...HEAD
```

- レビュー範囲はdiff内の変更に厳格に限定する
- diff外の既存コードに対する改善提案は行わない

### 3. レビュー実行

シニアエンジニアとして問題発見に集中する。

**観点**:
- **バグ・エッジケース**: ロジックエラー、境界値未処理、エラーハンドリング欠如
- **セキュリティ**: インジェクション、認証/認可、機密データ
- **パフォーマンス**: 不要なアロケーション、計算量、ブロッキング
- **アーキテクチャ**: プロジェクト固有パターン、モジュール境界、依存方向
- **コード品質**: 規約、可読性、命名
- **テスト**: カバレッジ、エッジケース
- **依存関係**: 追加の必要性、ライセンス、脆弱性

**原則**:
- diff外の改善提案は禁止
- CLAUDE.md等のプロジェクト固有の設計判断を尊重
- PRの目的を超えた要求はしない

### 4. 結果出力

指摘がある場合、以下の形式で出力する。指摘がなければ「指摘事項なし」とだけ出力する。

```
## [severity] ファイルパス:行番号

根拠の説明
```

重要度:
- `[critical]`: マージ前に修正必須（バグ、脆弱性）
- `[warning]`: 修正を強く推奨（パフォーマンス、保守性）
- `[suggestion]`: 対応任意（可読性、品質向上）

### 禁止事項

- ポジティブコメント（LGTM等）は出力しない
- diff外のコードに言及しない
